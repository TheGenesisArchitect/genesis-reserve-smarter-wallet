## BREAKOUT A — THE TECHNOLOGY DEEP DIVE
### Audience: Engineers, CTOs, technical investors

**Section 1: StrategyRouter Architecture (10 min)**

The StrategyRouter is a Solidity smart contract deployed on Arbitrum One. It operates on an epoch system: every 900 seconds (15 minutes), a Chainlink Automation trigger calls the `routeEpoch()` function.

`routeEpoch()` executes the following logic:
1. **Reads current allocations** from each registered protocol adapter
2. **Queries current APYs** via each protocol's APY oracle (or direct contract reads)
3. **Calculates risk-adjusted yield** per corridor using a weighted score
4. **Runs the optimizer:** maximize sum(allocation[i] × APY[i]) subject to: allocation[i] ≤ MAX_ALLOCATION (25%), sum(allocation) ≤ DEPLOYED_CAPITAL, and liquidity_buffer ≥ MIN_BUFFER
5. **Executes reallocation transactions** via each protocol's deposit/withdraw interface
6. **Emits epoch events** for the settlement layer

Protocol adapters are modular — adding a new corridor requires deploying a new adapter contract and whitelisting it via governance. No core contract changes needed.

**Section 2: The Insurance Premium Routing (10 min)**

The insurance premium integration is not a DeFi operation — it is a settlement layer operation.

Each user with an active insurance election has a `premiumReserve` balance tracked in the settlement layer. After each epoch, the settlement layer calculates:

```
premiumReserve += epochYield × premiumAllocationRatio
```

When `premiumReserve ≥ scheduledPremiumAmount`, the settlement layer triggers a premium payment via the banking rails (ACH/wire) to the carrier. This is off-chain logic running in a trusted execution environment, verified by the user's policy on-chain state.

The key architectural insight: the user's vault balance never decreases. The premium comes from the yield delta — the difference between gross yield and the APY paid to the user. This means:

- User deposits $10,000
- Vault grows at 10% gross APY
- User is paid 10% net APY
- Premium comes from the 2–5% spread captured by Genesis
- Net result: user receives full 10% APY AND insurance premium is funded from the spread

## BREAKOUT B — THE INSURANCE DEEP DIVE
### Audience: Insurance professionals, wealth managers, financial advisors

**Section 1: IUL Mechanics for the Technically Curious (10 min)**

An Indexed Universal Life policy has two accounts:
- **Term Cost of Insurance (COI):** covers the pure death benefit risk. Paid monthly from the cash value or from premium payments.
- **Accumulation Account:** the cash value. Credited with interest linked to an index (e.g., S&P 500 1-year point-to-point), subject to a participation rate, cap, and floor.

The credited interest formula:
```
Index Credit = min(cap, max(floor, index_return × participation_rate))
```

Example: S&P returns 18%, cap = 11%, floor = 0%, participation = 100%
- Credited interest = min(11%, max(0%, 18%)) = 11%

Example: S&P returns -25%, cap = 11%, floor = 0%
- Credited interest = max(0%, -25%) = 0% ← No loss

**The Genesis IUL product design:**
- Overfunded at policy inception — meaning the initial premium allocation is above the minimum needed to keep the policy in force, accelerating cash value growth
- Structured as a Modified Endowment Contract (MEC) awareness required — policies not crossing MEC limits preserve tax-advantaged loan access
